# 提示词模板：静态数据半工程化开发

请按以下约束实现，不要偏离：

1. 项目当前阶段不接数据库、不搭后端服务。
2. 使用 `public/data/*.json` 作为数据源。
3. 页面和图表组件不得直接读取 JSON 路径，必须通过 `src/api/dataService.ts` 访问。
4. `dataService` 的函数命名按业务拆分，如 `getOverview/getNodes/getClusters/getRelations/getPatterns/getTimeline/getSceneById`。
5. 所有数据先经过 `src/types` 类型定义和必要的适配函数，再传给 ECharts。
6. 代码结构要保证未来切换后端时，只改 `dataService` 内部路径（从 `/data/*.json` 切到 `/api/*`），页面层尽量不改。
7. 输出时优先给出：
   - 新增/修改了哪些文件
   - 数据契约（字段）
   - 如何验证（npm run dev / npm run build）

额外要求：
- 保持现有深色驾驶舱风格。
- 遵守 ESLint/TypeScript，不引入未使用变量。
- 如需 mock 数据，请按模块拆分 JSON，不要做一个超大单文件。
