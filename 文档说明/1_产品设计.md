# BigLotus CrowdEva 产品文档（基于现有实现）

面向智慧场馆的人群态势驾驶舱，当前代码基于 React 19 + TypeScript + Vite，UI 使用 Ant Design 6，图表使用 ECharts。主题为深色（主色 `#4cc3ff`，背景 `#070f1f`），数据以 Mock 生成，重点展示实时刷新与推演交互。

## 1. 信息架构与访问控制
- 路由：`/login`（登录） → `/dashboard/overview`、`/dashboard/monitor`、`/dashboard/prediction`、`/dashboard/simulation`。默认首页重定向到概览。
- 登录守卫：`src/router/index.tsx` 中的 `PrivateRoute`，本地 `localStorage` 存储 `isAuthenticated`。未登录访问受保护路由会跳转到 `/login`。
- 账号：硬编码 `admin/admin`（见 `src/pages/LoginPage.tsx`）。

## 2. 页面功能对照
### 2.1 登录页 `/login`
- 左侧为 LoginGraphic，右侧为表单。输入正确账号后写入 `localStorage` 并跳转仪表盘；错误弹出 `message.error`。

### 2.2 全域态势 `/dashboard/overview`
- 顶部 4 张 KPI 卡（总人数、拥堵指数、入场速率、出场速率），支持趋势标识。
- 主图：`CrowdDensityHeatmap`（分区 x 时间的热力矩阵，内置 6 个区域，近 60 分钟 10 分钟粒度）。
- 侧栏：`AlarmList` 列出近期告警，按等级 Tag 标记。

### 2.3 聚类监控 `/dashboard/monitor`
- 左侧：`ForceDirectedGraph` 动态力导向图（节点大小映射人流量，类别颜色对应安全/预警/危险），支持拖拽与点击；使用 `useDynamicData(generateForceGraphData, 3000)` 每 3 秒刷新。
- 右上：`NodeDetailPanel` 显示选中节点详情（随机生成男女比例、停留时间）。
- 右下：`SankeyFlowChart` 人群流向桑基图；点击力导向节点会触发 `generateSankeyData(nodeId)`，对相关连线缩放。

### 2.4 趋势研判 `/dashboard/prediction`
- 顶部：`ConfidenceBandChart`，包含历史实线、预测虚线与置信区间填充，数据由 `generatePredictionDataWithBounds` 生成，通过 `useDynamicData` 每 10 秒刷新。
- 底部：`ModelArena`，并列两个雷达图（LSTM、Transformer），自动比较综合得分并给出推荐与准确性差值。

### 2.5 应急沙盘 `/dashboard/simulation`
- 左侧：`SimulationGraph` 节点网络（由 `generateGraphData(generateZones())` 生成），力导向布局，可点击节点高亮。
- 右侧：`NodeControlPanel` 提供两类操作：
  - 关闭：将该节点出边改为虚线，并生成“下游中断”提示。
  - 限流：滑块 50%-200%，将下游节点标记虚线，并生成“通过率下降/排队增加”提示。
- 评估：`ImpactAssessmentPanel` 展示操作后的文本结果；支持“重置沙盘”恢复初始图数据。

## 3. 数据与组件
- Mock 生成：集中于 `src/utils/mockData.ts`（区域、热力、预测区间、模型分数、力导向、桑基、沙盘图数据等）。
- 动态刷新：`src/hooks/useDynamicData.ts` 基于 `setInterval` 周期调用生成器。
- 图表组件：`CrowdDensityHeatmap`、`ForceDirectedGraph`、`SankeyFlowChart`、`ConfidenceBandChart`、`ModelArena`/`ModelScoreRadar`、`SimulationGraph` 等，均使用 `echarts-for-react`。
- UI 组件：`KpiCard`、`AlarmList`、`NodeDetailPanel`、`NodeControlPanel`、`ImpactAssessmentPanel` 等，使用 Ant Design 卡片和基础组件。
- API 脚手架：`src/api/request.ts` / `src/api/index.ts` 预留 fetch 包装，当前页面全部使用 Mock 数据。

## 4. 主题与样式
- 全局主题：`src/router/index.tsx` 的 `ConfigProvider` 设置暗色算法与 token（主色、背景、文字、边框圆角等）。
- 布局样式：`src/layouts/MainLayout.tsx` + `MainLayout.css` 提供侧边菜单、Header 折叠、退出登录按钮。
- 全局样式：`src/index.css`、`src/App.css`，`panel-card` 用玻璃质感背景；深色下统一文字颜色。

## 5. 运行与构建
- 依赖：Node.js 18+。
- 命令：`npm install`、`npm run dev`、`npm run lint`、`npm run build`、`npm run preview`。
- 生产输出：`dist/`；部署与 Nginx 示例见 `DEPLOY.md` / `nginx.conf.example`，`deploy.sh` 为上传/重载示例脚本。

## 6. 后续对齐建议
- 接入真实后端：以 `mockData` 的返回结构为契约，替换生成器为 API 响应；必要时在 `useDynamicData` 或 `useApi` 中支持轮询/WebSocket。
- 数据校准：为 KPI、告警、桑基/力导向图添加实际指标映射与阈值配置。
- 交互增强：监控页节点/桑基联动高亮、沙盘模拟加入指标量化（如预计等待时间、容量利用率）。
