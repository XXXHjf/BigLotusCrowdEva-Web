# 趋势预测对比数据设计（静态 JSON 版）

## 1. 目标
用于“趋势预测”页面的双折线对比图：
1. 同一时间轴同时展示真实数据与预测数据。
2. 保留预测置信区间，但视觉上做适度收窄，避免喧宾夺主。

## 2. 文件位置（前端）
- 路径：`public/data/prediction/trend-comparison.json`

## 3. 数据结构
```json
{
  "points": [
    {
      "time": "19:00",
      "actual": 820,
      "predicted": 805,
      "lower": 770,
      "upper": 840,
      "extra": {
        "其他可能的字段": "字段值"
      }
    }
  ]
}
```

## 4. 字段说明
顶层字段：
- `points`：数组，按时间顺序排列。

`points[*]` 字段：
- `time`（必填，字符串）：时间刻度，建议 `HH:mm`。
- `actual`（必填，数字）：该时刻真实流量。
- `predicted`（必填，数字）：该时刻预测流量。
- `lower`（必填，数字）：预测置信区间下界。
- `upper`（必填，数字）：预测置信区间上界。
- `extra`（可选，对象）：保留附加字段（算法版本、样本量、误差分解等）。

## 5. 约束规则
1. 时间必须有序，不能乱序。
2. 每个时刻仅一条记录。
3. `lower <= predicted <= upper`。
4. `actual/predicted/lower/upper` 必须是数字。
5. 时间轴建议等间隔（例如每 5 分钟一个点）。

## 6. 前端展示约定
1. 真实数据：实线（主对比对象）。
2. 预测数据：虚线（与真实数据同轴对比）。
3. 置信区间：阴影保留，但进行“视觉收窄”。

说明：前端会基于 `lower/upper` 做收窄渲染（当前收窄系数约 `0.62`），
目的是增强真实 vs 预测的可读性，而非改变原始区间数据。

## 7. 替换数据时只需要做什么
1. 用你的真实结果覆盖 `points` 数组。
2. 保持字段名不变：`time/actual/predicted/lower/upper`。
3. 确保规则 `lower <= predicted <= upper` 成立。

完成后前端无需改代码。
