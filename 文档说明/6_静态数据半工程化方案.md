# 静态数据半工程化方案（当前项目执行版）

## 1. 结论
当前阶段可以不接数据库，不搭后端服务，采用 `public/data/*.json` 的静态数据方案。
但前端代码必须按“可替换为真实 API”的结构编写，避免后续返工。

一句话原则：
现在用 JSON，代码结构按未来可接后端来写。

## 2. 目标与边界
- 目标：快速展示师兄算法输出结果，支撑论文/汇报/演示。
- 边界：不做用户体系、不做持久化写入、不做复杂运维。
- 约束：图表展示层不直接依赖某个 JSON 文件路径，统一通过 `dataService` 获取数据。

## 3. 推荐目录
```text
project/
├─ public/
│  ├─ data/
│  │  ├─ overview.json
│  │  ├─ nodes.json
│  │  ├─ clusters.json
│  │  ├─ relations.json
│  │  ├─ patterns.json
│  │  ├─ timeline.json
│  │  └─ scenes/
│  │     ├─ scene_1.json
│  │     └─ scene_2.json
├─ src/
│  ├─ api/
│  │  └─ dataService.ts
│  ├─ pages/
│  ├─ components/
│  └─ types/
```

说明：
- JSON 放 `public/data`，构建后可直接通过 `/data/*.json` 访问。
- 页面和组件只调用 `src/api/dataService.ts`，不直接 `fetch('/data/xxx.json')`。

## 4. 代码约定（强制）
- 禁止在页面组件里写硬编码数据加载路径。
- 每类数据提供独立 service 方法，例如：`getNodes()`、`getClusters()`。
- service 返回统一结构（建议与现有 `ApiResponse<T>` 对齐），便于后续切换到 `/api/*`。
- 所有图表入参先经过 `src/types` 的类型约束和适配函数，再喂给 ECharts。

## 5. 最小实现模板
```ts
// src/api/dataService.ts
export async function getNodes() {
  const res = await fetch('/data/nodes.json')
  return res.json()
}

export async function getClusters() {
  const res = await fetch('/data/clusters.json')
  return res.json()
}
```

未来切换真实后端时，仅改 service 内部：
```ts
export async function getNodes() {
  const res = await fetch('/api/nodes')
  return res.json()
}
```

## 6. 数据契约建议
- 约定字段命名稳定（如 `id`、`name`、`timestamp`、`value`）。
- 每份 JSON 增加元信息：`version`、`generatedAt`、`source`。
- 时间字段统一 ISO 8601 字符串。
- 数值单位明确（人次、百分比、分钟等）。

示例：
```json
{
  "version": "v1",
  "generatedAt": "2026-03-15T10:00:00+08:00",
  "source": "python-pipeline-2026-03",
  "data": []
}
```

## 7. 迭代流程（建议）
1. 师兄输出算法结果（CSV/JSON）。
2. 你写一次转换脚本，生成规范化 JSON 到 `public/data`。
3. 前端仅验证契约与渲染，不改业务结构。
4. 后续若接后端，只替换 `dataService` 实现。

## 8. 风险与控制
- 风险：JSON 字段频繁变化导致页面反复改。
  - 控制：先冻结 `types` 与 JSON 契约，再开始批量做页面。
- 风险：数据量过大导致加载慢。
  - 控制：按页面拆 JSON，避免一个超大文件全量加载。
- 风险：多人改同一份 mock 数据冲突。
  - 控制：按模块拆分文件并加 `version`。

## 9. 本项目执行决定（2026-03-15）
- 当前阶段不接数据库。
- 当前阶段不强制后端服务。
- 采用静态 JSON + `dataService` 统一访问。
- 后续若演示要求升级，再将 `dataService` 对接真实 API。
