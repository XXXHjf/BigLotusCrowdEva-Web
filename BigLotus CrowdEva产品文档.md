作为产品经理兼设计师，我将不仅关注功能逻辑，更关注**视觉层级（Visual Hierarchy）与信息传达效率**。我们的目标是将一个“数据展示屏”升级为“智慧场馆指挥中枢”。

# BigLotus CrowdEva —— 智慧场馆人群态势感知系统

## 产品设计方案 (v2.0)

### 1. 设计理念与视觉系统

由于系统基于“聚类算法”而非“物理区域绑定”，我们的设计语言将从“拟物化（地图）”转向**“抽象数字化（Abstract Digital Twin）”**。

- **核心隐喻**：**声纳雷达 (Sonar Radar)**。我们将场馆视为一个圆形信号源，系统在不断扫描并捕捉人群形成的“信号块”。
- **色彩规范**：
  - **主色调**: `Tech Blue` (#1677FF - Ant Design 默认蓝，代表正常态)。
  - **背景色**: 推荐**深色模式 (Dark Mode)** 作为默认驾驶舱模式，背景使用深灰蓝 (#001529)，以突显发光的图表数据。
  - **预警色**:
    - `Safe Green` (#52C41A): 舒适流速。
    - `Warning Yellow` (#FAAD14): 密度升高，需关注。
    - `Danger Red` (#F5222D): 拥堵报警，伴随呼吸灯动效。
- **布局框架**: 采用 Ant Design Pro 的 `SideMenu` 布局，左侧导航，顶部为全局状态栏（显示时间、全局警报跑马灯）。

### 2. 路由架构 (Information Architecture)

将原有的单页 Dashboard 拆解为四个维度的子系统：

| **路径**                | **模块名称**          | **用户角色** | **核心价值**                               |
| ----------------------- | --------------------- | ------------ | ------------------------------------------ |
| `/login`                | 身份认证              | 全员         | 安全入口 (保持 `admin/admin` 逻辑)         |
| `/dashboard/overview`   | **全域态势 (驾驶舱)** | 指挥官       | **宏观视角**：当前场馆总体承载压力是多少？ |
| `/dashboard/monitor`    | **聚类监控 (实时)**   | 安保队长     | **微观视角**：具体哪一团人群出现了异常？   |
| `/dashboard/prediction` | **趋势研判 (预测)**   | 数据分析师   | **时间视角**：未来 1 小时走势如何？        |
| `/dashboard/simulation` | **应急沙盘 (演练)**   | 策划人员     | **决策视角**：限流措施是否有效？           |

### 3. 详细页面设计

#### 3.1 全域态势驾驶舱 (`/dashboard/overview`)

**设计目标**：一屏概览，秒级判断安全状态。

- **布局结构**：
  - **顶部 (Header)**: 4 个关键 KPI 卡片（总人数、拥堵指数、流入速率、流出速率）。
  - **中心 (Main)**: **极坐标雷达散点图**（视觉焦点）。
  - **右侧 (Sidebar)**: 实时流量仪表盘 + 报警列表。
- **核心图表 A：全景人群雷达 (Polar Radar Scatter)**
  - **图表原理**：利用后端聚类数据的坐标 $(x,y)$ 映射到极坐标系。
  - **视觉设计**：
    - **背景**：绘制 3-4 个同心圆环，代表场馆的“内场”、“看台”、“回廊”、“外围”。加上十字准星。
    - **数据点 (散点)**：每一个“气泡”代表一个后端计算出的人群聚类块。
    - **气泡大小**：映射该聚类的人数规模。
    - **气泡颜色**：映射拥堵等级（绿->红渐变）。
    - **动效**：添加一个逆时针旋转的“扫描扇区”阴影，模拟雷达实时探测的效果，增强科技感。
  - **交互**：鼠标悬停气泡，显示“Cluster #12：人数 520人，主要位于：北偏东 30°”。
- **核心图表 B：吞吐率仪表盘 (Dual Gauge)**
  - **位置**：右侧面板。
  - **设计**：两个半圆形仪表盘，分别显示 **Inflow (入场)** 和 **Outflow (散场)** 的每分钟人数。
  - **阈值线**：仪表盘上标记红色刻度线（设计最大通行力），指针超过红线即变红。

#### 3.2 聚类监控中心 (`/dashboard/monitor`)

**设计目标**：展示人群结构与相互关系，定位异常源头。

- **布局结构**：
  - **左侧 (Main)**: 占据 70% 宽度的**力导向拓扑图**。
  - **右侧 (List)**: “拥堵聚类 Top 5” 排行榜 + 详细参数面板。
- **核心图表：动态力导向拓扑图 (Force-Directed Cluster Graph)**
  - **图表原理**：
    - **节点 (Nodes)**：代表人群聚类块。节点越大，人数越多。
    - **边 (Edges)**：代表人群的移动/渗透趋势。如果两个聚类块之间有人快速移动，则建立连线。
  - **视觉设计**：
    - **物理模拟**：节点之间存在排斥力（避免重叠），不仅美观，而且能动态展示人群的分离与融合。
    - **颜色编码**：
      - 正常节点：半透明蓝色。
      - 拥堵节点：实心红色，并带有**波纹扩散 (Ripple Effect)** 动画，像是在求救。
  - **解决痛点**：完美规避了“没有精确底图”的问题，安保人员只需关注**变红且变大**的节点，以及它连向了哪里。
- **辅助图表：人群流向桑基图 (Sankey Diagram)**
  - **位置**：页面底部或模态框中。
  - **内容**：展示宏观流向，例如 `入口 A (2000人)` -> `东看台 (1500人)` / `VIP区 (500人)`。

#### 3.3 趋势研判分析 (`/dashboard/prediction`)

**设计目标**：展示时间维度的数据，建立对未来的信心。

- **核心图表 A：置信区间趋势图 (Confidence Band Line Chart)**
  - **设计**：
    - X 轴：时间（-1小时 到 +1小时）。
    - **实线**：历史真实数据。
    - **虚线**：AI 预测数据。
    - **阴影带 (Area)**：在预测虚线周围包裹一层半透明区域，代表“预测置信区间”（如 95% 可能性落在该范围）。这比单纯的一条线更具专业决策参考价值。
- **核心图表 B：模型竞技场 (Multi-Model Comparison)** [保持原设计]
  - **设计**：保留原来的多折线对比图。
  - **优化**：增加一个“准确率”评分板，实时计算当前哪个模型（LSTM vs Transformer）与真实数据的误差最小，自动推荐当前最可信的模型。

#### 3.4 应急沙盘演练 (`/dashboard/simulation`)

**设计目标**：提供“What-If”分析能力，手动干预系统。

- **交互逻辑设计**：
  1. **初始状态**：展示简化的**节点网络图**（基于上述拓扑图）。
  2. **用户操作**：
     - 点击某个关键节点（如“北门入口聚类”）。
     - 弹出操作面板：选择 **[关闭通道]** 或 **[限流 50%]**。
  3. **系统反馈**：
     - 图表瞬间刷新，展示**“推演后的状态”**。
     - 受影响的节点（如下游的看台）变成虚线或改变颜色。
     - **影响评估面板**弹出：*"警告：该操作将导致‘东看台入口’排队时间增加 10 分钟。"*

### 4. 开发实施与数据策略

为了保证开发效率和体验，遵循以下规范：

1. **Mock/Real 双模式切换**:
   - 在 `src/utils` 下封装统一的 DataLayer。
   - 全局配置 `IS_DEMO_MODE`。
   - **Mock 模式**: 使用 `setInterval` 生成带有正弦波动的数据，模拟真实的人潮呼吸感，避免图表静止不动。
   - **Real 模式**: 对接 `src/api/crowd.ts`。
2. **响应式布局**:
   - 使用 Ant Design 的 `Grid` 系统 (`Row/Col`)。
   - 在大屏 (1920px) 下展示 3 列布局，在笔记本 (1366px) 下自动折叠为 2 列。
3. **性能优化**:
   - ECharts 实例需在组件卸载时 (`useEffect cleanup`) 及时销毁 (`dispose`)，防止内存泄漏。
   - 对于高频更新的雷达图，开启 ECharts 的 `large: true` 优化选项（如果点非常多）。

### Phase 4: 聚类监控中心 (`/monitor`) —— 核心攻坚

当前状态：空页面/占位符。

核心目标：实现“微观视角”的关联分析，回答“拥堵源头在哪里？人群流向了哪里？”。

#### 1. 页面布局设计

- **布局模式**：`7:3` 左右分栏。
- **左侧 (70%)**：**力导向拓扑关系图 (Force-Directed Graph)** —— 视觉重心。
- **右侧 (30%)**：
  - **上半部分**：选定节点的**详情透视镜**（显示该聚类的具体画像，如男女比例、平均停留时间）。
  - **下半部分**：**人群流向桑基图 (Sankey Diagram)**（宏观流向）。

#### 2. 核心图表 A：力导向拓扑图 (Force-Directed Graph)

这是本页面的灵魂。不同于首页雷达图的“方位感”，这里强调的是“关系感”。

- **ECharts 配置思路**：
  - `series.type`: `'graph'`
  - `layout`: `'force'` (开启力引导布局)。
  - **节点 (Nodes)**：
    - 代表“聚类块”。
    - `symbolSize`: 根据 `capacity` (人数) 动态映射大小 (e.g., 20px - 80px)。
    - `itemStyle.color`: 动态映射拥堵等级（绿/黄/红）。**红色节点**建议开启 `effectScatter` 类似的呼吸动效（可通过叠加图层实现）。
  - **边 (Edges)**：
    - 代表“渗透关系”。
    - `lineStyle.width`: 根据流量大小决定粗细。
    - `lineStyle.curveness`: 设置为 `0.3`，使连线呈优美的曲线，避免直线僵硬。
  - **物理引擎 (Force)**：
    - `repulsion`: 设为较大值 (如 500)，让节点尽量散开，模拟人群保持社交距离。
    - `gravity`: 设为较小值 (0.1)，让它们向画面中心微弱聚集，不飞出屏幕。
- **交互设计**：
  - 支持**拖拽 (Draggable)**：允许指挥官手动把纠缠在一起的节点拉开，梳理关系。
  - **点击联动**：点击某个节点，右侧详情面板和桑基图仅展示与该节点相关的数据。

#### 3. 核心图表 B：流向桑基图 (Sankey Diagram)

- **设计目的**：弥补拓扑图“看不出方向”的短板。
- **ECharts 配置思路**：
  - **层级 (Levels)**：定义三列 —— `来源 (Inlets)` -> `通道 (Corridors)` -> `目标区 (Destinations)`。
  - **色彩**：使用半透明渐变色，营造“水流”的质感。



### Phase 5: 趋势研判分析 (`/prediction`) —— 深度赋能

当前状态：空页面/占位符。

核心目标：引入“时间轴”，展示 AI 预测能力。

#### 1. 页面布局设计

- **布局模式**：上下结构。
- **顶部 (60%)**：**置信区间趋势图 (Confidence Band Line)**。
- **底部 (40%)**：**模型准确率竞技场 (Model Arena)**。

#### 2. 核心图表 A：置信区间趋势图

这是体现“专业性”的关键。普通的折线图太单薄，带置信区间的图表才像数据科学产品。

- **图表构成**：需要 3 个 Series 叠加。
  1. **实线 (Solid Line)**：历史真实数据（00:00 - 当前时刻）。颜色：亮蓝。
  2. **虚线 (Dashed Line)**：AI 预测均值（当前时刻 - 未来1小时）。颜色：青色。
  3. **区域带 (Area/Band)**：预测的上下界（Upper bound / Lower bound）。
     - 使用 `type: 'line', stack: 'confidence'` 技巧，或者使用 `areaStyle` 填充两条线之间的区域。
     - 颜色：半透明青色 (`opacity: 0.2`)。
- **视觉隐喻**：
  - 阴影越窄，说明 AI 对预测越有信心；阴影越宽，说明未来不确定性大。

#### 3. 核心图表 B：模型准确率竞技场

- **设计**：
  - 左侧显示两个雷达图或仪表盘，分别代表 `LSTM` 和 `Transformer` 两个模型的当前评分。
  - 评分维度：`准确性`、`响应速度`、`抗噪能力`。
  - 系统自动高亮推荐的模型：“当前场景推荐使用 **Transformer**，准确率高出 12%”。



### Phase 6: 系统级优化与“最后一公里”

完成上述两个页面后，建议进行以下系统级打磨：

1. **数据层重构 (Data Layer)**:
   - 目前的 `mockData.ts` 可能是静态的。
   - 建议编写一个 **WebSocket 模拟器** 或 **轮询 Hook** (`useInterval`)，让 `/monitor` 里的力导向图节点真的在动，让 `/overview` 的雷达点在闪烁。动态是“指挥中心”的生命力。
2. **深色模式微调 (Dark Mode Polish)**:
   - 检查 Ant Design 的组件（Table, Card, Modal）在深色背景下是否有白边或对比度不足的问题。
   - 确保所有 ECharts 的文字颜色 (`textStyle.color`) 都是浅灰或白色，而非默认的深灰。
